What is Git?

  • Tool for storing a history of changes, publishing those changes, and collaborating with others on making changes
  • Can be used with any files, but most commonly used with source code

What is GitHub?

  • Service that helps developers publish and collaborate on code
  • Can only be used with Git

Git Setup

  2. Use the installer at git-scm.com/download/mac
  3. Download and unzip the example project
  4. Open Terminal
  5. Use cd to navigate to the unzipped project
  6. Set your Git name: git config --global user.name "Your Name"
  7. Set your Git email address: git config --global user.email your-email-address@example.com

Starting a Project

  2. Create a new Git repository: git init
  3. Check the current status: git status
  4. Mark files: git add common-area.jpg event-goal.txt
  5. Check the current status: git status
  6. Store the marked files: git commit
  7. Review: git log

Improving Another Project (Part 1)

Improving Another Project (Part 2)

  2. Correct my-project.txt

  5. Check the current status: git status
  6. Store the marked file: git commit
  7. Review: git log
  8. Publish your improvements: git push origin master
  9. Submit a pull request

Accepting an Improvement

  2. On GitHub, visit your project's Pull Requests page
  3. Select the open pull request
  4. Review the improvements
  5. If you like the improvements, click Merge pull request
  6. Download the improvement: git pull origin
